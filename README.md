# Вторая память

Ассистент в одно окно: пишете обычным русским языком — он сам понимает, что это (долг, встреча, задача или заметка), раскладывает по полочкам и отдаёт по первому запросу.

## Зачем

> «Мне хочется в одно окно иметь ассистента, которому я говорю: запиши какую-то встречу… Как вторая память, которая будет автоматически всё распределять. И в любую секунду через одно окно я хочу получать все данные. Например: отгрузи мне все долговые обязательства заказчиков, которые на сегодняшний день состоят».

Ровно это здесь и сделано. Один чат — на запись и на выдачу. Никаких форм, категорий и меню: категорию, сумму, контрагента и срок ассистент вытаскивает из фразы сам.

## Быстрый старт

Нужен только Node.js 18.17 или новее. Ни одной npm-зависимости.

```bash
git clone git@github.com:meloch287/second-memory.git
cd second-memory
node src/server.mjs
```

Откройте http://localhost:8790 — и пишите.

## Что он понимает

### Записи

| Вы пишете | Что происходит |
|---|---|
| `Иванов должен 50 000 до 20 июля` | Долг: Иванов → вам, 50 000 ₽, срок 20.07 |
| `я должен Петрову 15к до пятницы` | Долг: вы → Петрову, 15 000 ₽, срок — ближайшая пятница |
| `Заказчик «Ромашка» должен 120 000 до конца месяца` | Долг: Ромашка, 120 000 ₽, срок — последний день месяца |
| `запиши встречу с Сергеем завтра в 15:00` | Встреча, завтра 15:00 |
| `созвон с командой в пятницу в 10:00` | Встреча, пятница 10:00 |
| `напомни оплатить хостинг через 3 дня` | Задача со сроком |
| `не забыть отправить акт до 15.07` | Задача со сроком |
| `клиент Альфа просил расширить договор` | Заметка |

Фраза, не похожая ни на что из перечисленного, сохраняется заметкой — ничего не теряется.

### Запросы

| Вы спрашиваете | Ответ |
|---|---|
| `отгрузи все долговые обязательства заказчиков` | Все открытые долги: кто, сколько, до какого числа, итоги по направлениям, просрочка помечена |
| `сколько мне должны` | Сумма одной строкой: «Вам должны: 50 000 ₽ (1 долг)» |
| `что у меня завтра` | Сводка на день: встречи, задачи, долги со сроком |
| `покажи встречи на неделю` | Встречи ближайших 7 дней |
| `сводка` | Общая картина: счётчики, итоги по долгам, ближайшие встречи и задачи |

### ИИ: саммари и вопросы по базе

Подключите любой OpenAI-совместимый LLM (по умолчанию [polza.ai](https://polza.ai), модель `google/gemini-2.5-flash`) — и появляются два режима:

| Вы пишете | Что происходит |
|---|---|
| `саммари` | ИИ получает всю базу (долги, встречи, задачи, заметки + последние сообщения диалога — мини-RAG) и делает деловое саммари: кто сколько должен, что горит, что рекомендуется |
| любой вопрос со знаком `?` | ИИ отвечает по вашим данным: «у кого из должников самый горящий срок?» → «Петров. Срок 10.07.2026» |

Ключ задаётся в `.env` (см. [.env.example](.env.example)):

```bash
cp .env.example .env   # впишите AI_API_KEY
node src/server.mjs
```

Без ключа всё остальное работает как раньше — правила не зависят от ИИ.

### Команды

| Команда | Действие |
|---|---|
| `готово 3` | Закрыть запись №3 (долг оплачен, задача сделана) |
| `удали 5` | Удалить запись №5 |
| `очистить чат` | Стереть историю переписки (записи в памяти не трогаются) |
| `помощь` | Все примеры прямо в чате |

### Даты и суммы

Понимает: `сегодня`, `завтра`, `послезавтра`, `через 3 дня`, `через неделю`, `в пятницу`, `20 июля`, `15.07`, `20.07.2026`, `до конца месяца`, `до конца года`, время `в 15:00` и `в 7 вечера`. Суммы: `50 000`, `15к`, `1,5 млн`, `120 000 руб`.

## Интерфейс

Одно окно: чат слева, справа карточка «Память» со шкалой наполненности - видно, хватает ли ассистенту данных для уверенных ответов (пустая база - шкала на нуле). Под шкалой статус и счётчики: записи, факты, дни. Каждый ответ ИИ на вопрос сопровождается мини-шкалой «Опора на память»: сколько релевантного нашлось в базе именно под этот вопрос. Кнопка «Меню» с тремя действиями (Саммари, Очистить чат с подтверждением, Помощь), мягкие анимации появления сообщений.

Интерфейс собран по WCAG 2.2 AA: семантические лендмарки, корректные live-регионы чата (ответ ассистента объявляется скринридером, своё сообщение — нет), полная работа с клавиатуры (включая roving tabindex по плеерам голосовых), видимый фокус, контраст всех пар цветов проверен расчётом, уважается `prefers-reduced-motion`.

## Голосовые сообщения

Как в Telegram: нажали микрофон — идёт запись с живой волной, пульсирующей точкой и таймером. «Отправить» — голосовое появляется в чате пузырём с плеером (волна, play/pause, длительность), «Отмена» или Escape — запись выбрасывается. Лимит записи — 2 минуты.

Главное — транскрибация: пока идёт запись, речь параллельно распознаётся прямо в браузере (Web Speech API, без внешних ключей и серверов). Расшифровка показывается под волной в самом пузыре и тут же уходит ассистенту как обычная фраза. Сказали голосом «Иванов должен пятьдесят тысяч до двадцатого июля» — получили записанный долг с суммой и сроком.

Детали:

- Аудио живёт только в вашем браузере (blob в рамках сессии) — на сервер уходит лишь текст расшифровки. Приватность по умолчанию.
- Расшифровка в пузыре — это и текстовая альтернатива аудио (WCAG 1.2.1).
- Если браузер не поддерживает распознавание речи (например, Safari), голосовое запишется и проиграется, но в пузыре будет честная пометка «Расшифровка недоступна».
- Если доступ к микрофону запрещён — понятная ошибка под композером с инструкцией, что включить.

Конвейер «голос → текст → структурированная запись» подсмотрен у [Voicenotes](https://voicenotes.com/) и [AudioPen](https://www.audiopen.ai/), UX плеера — у Telegram ([голосовые с волной и расшифровкой](https://core.telegram.org/api/transcribe)).

## Telegram-бот: друг, а не секретарь

Бот в Telegram - это цифровой дневник с характером. Получите токен у [@BotFather](https://t.me/BotFather), впишите `TELEGRAM_BOT_TOKEN` в `.env` и запустите сервер.

Как он работает:

- **Знакомство.** На `/start` бот не показывает меню. Сначала он просит придумать имя ему самому, потом спрашивает ваше имя, ритм жизни и главную цель. Всё попадает в профиль: дальше он общается лично, на «ты», откликается на своё имя и иногда подписывается им.
- **Повторный `/start` ничего не сбрасывает.** Бот просто здоровается одной из двадцати живых фраз: «О, ты вернулся! Как день прошёл?»
- **Просто пишите как в дневник.** «Планёрка прошла жёстко», «клиент должен 50 000 до пятницы», «завтра созвон в 10:00». Долги, встречи и задачи бот тихо раскладывает в структурированную базу, остальное запоминает как есть.
- **Двойная база.** Всё написанное падает в сырую базу (БД №1). Фоновый worker раз в 3 минуты прогоняет новое через ИИ и превращает в факты: люди, темы, эмоции (БД №2). Эти факты читает RAG при каждом ответе - бот помнит, о чём вы говорили неделю назад.
- **Четыре команды.** `/summary` - итоги дня по главам (🕒 Утро, 💼 День, 🌙 Вечер, 💡 Инсайт) с кнопками «Расскажи подробнее» и «Что завтра?». `/help` - тёплая подсказка с примерами. `/start` - поздороваться. `/reset` - стереть память друга и познакомиться заново: бот переспросит («уверен, что хочешь убить меня?((»), после подтверждения сотрёт по этому пользователю всё: профиль, разговоры, факты и его записи.
- **Медиа понимает.** Голосовые, mp3 и аудиофайлы расшифрует и обработает как текст. Фото и статичные стикеры разглядит и запомнит описание, у гифок посмотрит превью. Пока думает - честно показывает «печатает…».
- Long polling - белый IP и вебхуки не нужны. Веб и бот делят одну базу.

Без ключа ИИ бот честно отвечает правилами и ничего не теряет.

## Деплой 24/7

Бот живёт на сервере как systemd-сервис (`node src/server.mjs`). Если сервер в России, Telegram API недоступен напрямую - решение: xray-клиент с vless-узлом за границей отдаёт локальный HTTP-прокси, а Node 24 умеет ходить через него нативно:

```ini
Environment=NODE_USE_ENV_PROXY=1
Environment=HTTPS_PROXY=http://127.0.0.1:10809
Environment=NO_PROXY=api.polza.ai,api.gonkagate.com,localhost,127.0.0.1
```

Так к Telegram бот ходит через прокси, а к ИИ-провайдерам - напрямую. Кода это не касается вообще.

## Obsidian: карта памяти

```bash
npm run obsidian
```

Команда собирает из базы готовый Obsidian-vault (`data/obsidian`): заметки по людям, дням и темам, связанные [[ссылками]]. Откройте папку как vault и включите Graph View - связи «кто, когда и с чем упоминался» нарисуются сами, как карта мыслей.

## HTTP API

| Метод | Путь | Описание |
|---|---|---|
| `POST` | `/api/message` | Тело `{"text": "..."}` — та же обработка, что и в чате. Ответ: `{"reply": "...", ...}` |
| `GET` | `/api/entries?type=debt&status=open` | Записи с фильтрами по типу (`debt`/`meeting`/`task`/`note`) и статусу (`open`/`done`) |
| `GET` | `/api/digest` | Сводка на сегодня |
| `GET` | `/api/memory-stats` | Наполненность памяти: `{score, level, label, counts}` |
| `POST` | `/api/history/clear` | Очистить историю диалога (контекст ИИ) |
| `GET` | `/api/health` | `{"ok": true}` |

Пример:

```bash
curl -s -X POST localhost:8790/api/message \
  -d '{"text":"Иванов должен 50 000 до 20 июля"}'
```

## Как устроено

```
src/server.mjs    HTTP-сервер: статика + JSON API (node:http), загрузка .env
src/brain.mjs     логика: действие над хранилищем + ответ по-русски
src/ai.mjs        ИИ-слой: персона друга, саммари, факты, RAG-контекст, расшифровка аудио
src/worker.mjs    фоновая суммаризация: сырые записи -> факты для RAG
src/obsidian.mjs  экспорт памяти в Obsidian-vault со ссылками для графа
src/parser.mjs    классификация фразы: долг / встреча / задача / заметка / запрос / команда
src/dates.mjs     извлечение русских дат, времени и сумм (регулярки без \b — он не работает с кириллицей)
src/store.mjs     JSON-хранилище с атомарной записью (tmp + rename)
src/telegram.mjs  Telegram-бот (long polling), включается токеном
public/           веб-интерфейс: index.html + styles.css + app.js, без фреймворков
test/             26 тестов на node:test
```

Разбор фразы — детерминированные правила, а не LLM: работает мгновенно, офлайн, бесплатно и предсказуемо. Точка расширения — `handleMessage()` в `src/brain.mjs`: сюда легко добавить вызов любой LLM для свободных формулировок, сохранив правила как быстрый путь.

## Хранение

Всё лежит в одном файле `data/memory.json` (путь меняется переменной `SM_DATA`). Формат записи:

```json
{
  "id": 1,
  "type": "debt",
  "counterparty": "Иванов",
  "amount": 50000,
  "direction": "in",
  "due": "2026-07-20T00:00:00.000Z",
  "status": "open",
  "text": "Иванов должен 50 000 до 20 июля",
  "createdAt": "2026-07-06T09:00:00.000Z"
}
```

`direction: "in"` — должны вам, `"out"` — должны вы. Резервная копия — просто копия файла.

## Ограничения — честно

- Разбор построен на правилах: экзотические формулировки могут уйти в заметки (и остаться доступными поиском), имена сохраняются в той форме, в какой написаны («Петрову», не «Петров»).
- Хранилище однопользовательское, авторизации нет — это личный инструмент для localhost или закрытого контура. Не выставляйте порт в интернет как есть.
- Транскрибация голосовых в браузере зависит от Web Speech API (в Chrome работает, в Safari — нет). В Telegram голосовые расшифровывает ИИ — но только при заданном `AI_API_KEY`.
- ИИ-саммари и ответы на вопросы требуют внешний LLM-API (ключ в `.env`); без него ядро полностью работоспособно на правилах.
